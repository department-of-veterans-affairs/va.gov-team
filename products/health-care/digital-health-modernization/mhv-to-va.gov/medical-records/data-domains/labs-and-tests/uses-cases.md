# Labs and Tests Use Cases
**Last updated:** Dec 16, 2022

For all of these use cases, the user must have an MHV account and be logged in, if the user is not logged in, the only thing they can access in the Labs and Tests section is [describe] unauth content advising them that they must log in to view their labs and tests.

## Common use cases
### User is not logged in
- User will need to log in with one of the accepted methods.  Once logged in, they can select the naviation option for labs and tests, and will be presented with a landing page.

### User is logged in
- When a a user is logged in and selects the naviation option for labs and tests, they will be presented with a landing page.
