# Working with MHV Accessibility
Different ways to connect and work with the MHV Accessibility team.

Last updated: 9 Jan 2026 (Note that this is an in-progress, living document)

## Requesting accessibility team support

Please feel free to use any or all of the following channels to engage accessibility support:

- Use the @mhvaccessibility Slack user group to request support, for example, for staging reviews.
- Post questions and support requests in the [MHV Accessibility Slack channel](https://dsva.slack.com/archives/C05QG2H510T).
- Assign tickets to the “MHV - Accessibility” project. For tickets needing accessibility review, set the status to “QA / A11y review”. Otherwise, set the status to “No status” or “Inbox”.
- Tag @sarahhortonadhoc and/or @steveg-IAT in comments or in the ticket body.

## Reporting accessibility issues

- Use the [MHV Accessibility Issue template](https://github.com/department-of-veterans-affairs/va.gov-team/issues/new?template=mhv-a11y-issue.yaml).
- Confirm that the ticket has the “accessibility” label.
- Add the “MHV - Accessibility” project to the ticket .
- Set the status to “Inbox”.
  
## Collaborating on addressing accessibility issues
  
- Monitor status of tickets assigned to the accessibility team on the [MHV - Accessibility project board](https://github.com/orgs/department-of-veterans-affairs/projects/1898/views/8).
- Include the accessibility team in any standups or ceremonies that may include accessibility-related tickets.
- Assign tickets to the “MHV - Accessibility” project.
- Assign tickets to @sarahhortonadhoc (for UX) or @steveg-IAT (for engineering).
