# Payment History Error Discovery

In The Beginning, there was the user story **Unable to view payment history** [#119795](https://github.com/department-of-veterans-affairs/va.gov-team/issues/119795). It was created to investigate reports of users not being able to see their payments on the *Your VA Payments* page. It was found that all exceptions were being swallowed when calling the BGS PaymentService, so when that happens, the user simply sees that they have no payments (whether that's true or not). Without seeing the actual exceptions that were causing the errors, it was determined that at least a certain subset of reported cases were caused by some temporary system or network outage (likely the former). A user story to improve exception logging was created to get better details, **Improve exception logging on the BGS::PaymentService** [#120937](https://github.com/department-of-veterans-affairs/va.gov-team/issues/120937)

This last story was picked up at the same time as the story **Migrate Sentry Logging to Vets::SharedLogging - Payment History, Letters and Disability Ratings** [#123972](https://github.com/department-of-veterans-affairs/va.gov-team/issues/123972) was created to address a platform initiative to migrate away from Sentry logging. By investigating the behavior of the `log_exception_to_sentry` method, it was discovered that, in fact, we had been logging detailed exception messages all along. Doh! With some help from AI, I was able to build a query to gather a random sample of exceptions that were associated with the "BGS::PaymentService returned nil" message.
